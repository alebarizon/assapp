## TEXTO PARA SUBMISSÃO

Ao final da Fase 1, o projeto prevê a entrega dos seguintes resultados, organizados por categoria:

**Resultados científicos**

R1 — Framework de requisitos para arquitetura de memória institucional em sistemas AMS: especificação formal dos requisitos funcionais e de modelagem de dados necessários para preservação ativa de conhecimento institucional em organizações com liderança rotativa. Este será o primeiro framework do tipo publicado para o contexto de associações científicas brasileiras, contribuindo para a literatura de Sistemas de Informação e Gestão do Conhecimento.

R2 — Modelo de interface adaptativa para gestores voluntários: relatório técnico com os resultados dos testes A/B comparando interface adaptativa e interface padronizada em contexto de uso ocasional. Contribuição original ao campo de User Modeling, Adaptation and Personalization (ACM UMAP), que não dispõe de estudos neste perfil de usuário.

R3 — Esquema de grafo temporal de engajamento em associações científicas: especificação formal do modelo de dados orientado a grafos, com definição de nós (pesquisadores, eventos, publicações), arestas tipadas (filiação, submissão, avaliação, participação) e timestamps. Acompanhado dos resultados preliminares de predição de links (AUC-ROC) obtidos com os dados piloto da ABCiber — primeiro dataset longitudinal do tipo disponível para associações científicas brasileiras.

R4 — Artigo científico submetido a periódico nacional indexado (Perspectivas em Ciência da Informação, Qualis A1, ou equivalente), reportando os resultados de R1 e R3.

**Resultados tecnológicos**

R5 — Protótipo funcional da plataforma SaaS implantado com a ABCiber: sistema operacional cobrindo os módulos de gestão de membros/filiações, gestão financeira integrada, gestão de eventos científicos com submissão de trabalhos e avaliação por pareceristas, e módulo de memória institucional com histórico auditável por ciclo de mandato. O protótipo operará em ambiente de produção real com usuários da ABCiber ao longo da Fase 1, garantindo condições realistas de coleta de dados para as hipóteses H1, H2 e H4.

R6 — Base de dados de grafo temporal (dataset piloto): conjunto de dados estruturado das interações dos pesquisadores filiados à ABCiber com a plataforma ao longo de pelo menos um ciclo completo de evento científico, formatado para análise em ferramentas de ciência de redes (NetworkX, Neo4j ou equivalente). Este dataset constituirá o ativo de pesquisa mais relevante para a continuidade do projeto na Fase 2.

**Resultados de mercado**

R7 — Validação de modelo de negócio com clientes piloto adicionais: além da ABCiber, o projeto buscará implantar versões beta da plataforma em pelo menos duas outras associações científicas paulistas durante a Fase 1, validando o encaixe produto-mercado (product-market fit) e refinando o modelo de assinatura SaaS.

R8 — Lean Canvas atualizado e documentação de propriedade intelectual: atualização do modelo de negócio com dados reais de uso, definição das funcionalidades patenteáveis ou registráveis como software (INPI), e mapeamento de potenciais parceiros estratégicos para a Fase 2 (aceleradoras, investidores-anjo, programas de inovação de associações de classe).
